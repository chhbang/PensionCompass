요청하신 2-1 항목(매도 대상 상품 선택 및 리밸런싱 시점/만기 예약 여부 선택 UI)은 실무적으로 매우 유용하고 중요한 기능입니다. 특히 만기가 있는 상품을 다룰 때 예약 기능을 분리하고, 사용자의 매도 자율성을 보장하는 UI는 포트폴리오 관리에 필수적입니다.

추가된 요구사항을 모두 반영하여 SLifeIrpRebalancer 앱의 상세 설계서를 새롭게 업데이트해 드립니다.

# **SLifeIrpRebalancer 앱 상세 설계서 (업데이트 버전)**

## **1\. 프로젝트 개요**

* 프로젝트명: SLifeIrpRebalancer  
* 개발 환경: Visual Studio 2026  
* 플랫폼/프레임워크: Windows 11, WinUI 3 (Blank App, Packaged),.NET 10 (또는.NET 9\)  
* 언어: C\#, XAML  
* 핵심 목적: 삼성생명 IRP 상품 목록(HTML) 파싱, 사용자 계좌 상태 입력, 매도 조건 설정, AI(Gemini/GPT/Claude) 기반 리밸런싱 포트폴리오 제안 및 PDF 출력 기능 제공.

## **2\. 시스템 아키텍처 및 기술 스택**

* 아키텍처 패턴: MVVM (Model-View-ViewModel) 패턴 (CommunityToolkit.Mvvm 라이브러리 활용)  
* HTML 파싱: HtmlAgilityPack 라이브러리를 사용하여 삼성생명 HTML 파일의 DOM 트리를 분석.  
* PDF 생성: QuestPDF 라이브러리 사용 (C\# 환경에서 선언적 방식으로 깔끔한 PDF 레이아웃 생성).  
* AI API 통신: C\# 기본 HttpClient를 활용한 REST API 통신 및 System.Text.Json을 통한 JSON 직렬화/역직렬화.

## **3\. 데이터 구조 설계 (Data Models)**

### **3.1 원리금보장형 상품 CSV 스키마**

| 컬럼명 | 타입 | 설명 |
| :---- | :---- | :---- |
| 운용사 | string | 금융회사명 (삼성생명, 저축은행 등) |
| 상품코드 | string | 고유 코드 |
| 상품명 | string | 원리금보장형 상품 이름 |
| 금리 | string | 적용 연이율 (예: 3.45%) |
| 만기기간 | string | 만기 (예: 1년, 3년, 5년) |

### **3.2 펀드 상품 CSV 스키마**

| 컬럼명 | 타입 | 설명 |
| :---- | :---- | :---- |
| 운용사 | string | 자산운용사명 (예: 삼성자산운용, 미래에셋 등) |

| 수익률(3개월) | string | HTML에서 판별된 3개월 수익률 |
| 수익률(6개월) | string | HTML에서 판별된 6개월 수익률 |
| 수익률(1년) | string | HTML에서 판별된 1년 수익률 |
| 수익률(3년) | string | HTML에서 판별된 3년 수익률 |

### **3.3 사용자 계좌 상태 및 매도 설정 모델 (AccountStatusModel.cs)**

* TotalAmount (decimal) \- 총 적립금 (필수)  
* DepositAmount (decimal?) \- 입금액 (선택)  
* ProfitAmount (decimal?) \- 운용수익 (선택)  
* RebalanceTiming (enum) \- 리밸런싱 시점 (Immediate: 즉시 일반 리밸런싱, MaturityReservation: 만기 예약용 리밸런싱)  
* OwnedItems (List) \- 보유 상품 리스트

*보유 상품 세부 모델 (OwnedProductModel.cs)*

* ProductName (string) \- 상품명 (CSV 기반 자동완성 연동)  
* CurrentValue (decimal) \- 보유 적립금 (필수)  
* ReturnRate (decimal?) \- 현재 수익률 (선택)  
* AnnualizedReturn (decimal?) \- 연환산 수익률 (선택)  
* InvestedDays (int?) \- 운용 일수 (선택)  
* TotalShares (decimal?) \- 좌수 (선택)  
* SellAction (enum) \- 매도 방식 (FullSell: 전량 매도, PartialSell: 부분 매도, Keep: 유지)  
* SellAmount (decimal?) \- 부분 매도 시 직접 입력한 매도 금액

## **4\. UI/UX 화면 설계**

앱은 좌측 네비게이션(NavigationView)을 통해 총 5개의 메인 화면으로 구성됩니다.

### **화면 1: 환경 설정 (Settings)**

* 기능: AI 서비스 선택(Gemini, Claude, GPT) 및 API Key 입력 폼.  
* 종신형 연금 옵션: "연금 개시 시 종신형 수령 예정 (삼성생명 운용사 상품으로 제한)" 체크박스 제공.

### **화면 2: 상품 데이터 준비 (Data Preparation)**

* 기능: HTML 파일 업로드 및 CSV 변환.  
* UI 요소:  
  * "HTML 파일 불러오기" 버튼.  
  * 파싱 진행률을 보여주는 ProgressBar.  
  * 결과를 미리 볼 수 있는 2개의 DataGrid 탭 (원리금보장형 탭, 펀드 탭).  
  * "CSV로 저장" 버튼.

### **화면 3: 내 계좌 정보 (My Account)**

* 기능: 현재 IRP 계좌의 총 잔고 및 보유 상품의 기초 정보 입력.  
* UI 요소:  
  * 상단: 총 적립금(필수), 입금액, 운용수익 입력 텍스트 박스.  
  * 중단: AutoSuggestBox를 통해 화면 2의 CSV 데이터를 기반으로 보유 상품 검색 및 추가.  
  * 하단: 추가된 상품 목록을 보여주는 DataGrid. 각 행마다 적립금(필수)과 수익률, 좌수 등의 부가 정보 기입.

### **화면 4: 매도 대상 및 리밸런싱 시점 설정 (Sell Target & Timing) (신규 추가)**

* 기능: 리밸런싱의 성격을 정의하고, 화면 3에서 입력된 각 상품에 대한 매도 지침을 사용자로부터 입력받습니다.  
* UI 요소:  
  * 상단 시점 선택: 라디오 버튼(RadioButtons)을 배치하여 "최대한 즉시 일반 리밸런싱 진행"과 "만기 예약용 리밸런싱 진행" 중 하나를 묻고 필수 선택하게 합니다.  
  * 하단 매도 설정: 화면 3의 상품 목록이 다시 표시되며, 각 행마다 ComboBox를 두어 '전량 매도', '부분 매도', '유지(매도 안함)'를 선택하게 합니다.  
  * 금액 입력 제어: '부분 매도'를 선택한 행에 한해서만 매도할 금액을 기입할 수 있는 TextBox가 활성화됩니다.

### **화면 5: AI 포트폴리오 리밸런싱 (AI Rebalance)**

* 기능: 입력된 모든 정보를 종합하여 AI 제안을 받고 PDF로 출력.  
* UI 요소:  
  * 사용자 추가 요구사항 입력 텍스트 영역 (예: "이란 전쟁 상황 반영해줘").  
  * "포트폴리오 제안 받기" 버튼 및 로딩 스피너.  
  * AI 응답 결과를 렌더링하는 RichTextBlock 또는 웹뷰(마크다운 렌더러).  
  * "PDF로 내보내기" 버튼.

## **5\. 핵심 모듈 설계 (로직)**

### **5.1 HTML 파서 및 운용사 추출 로직**

* HtmlAgilityPack을 이용해 테이블을 순회하며 수익률 기간 컬럼을 동적으로 헤더에 반영합니다.  
* 운용사 식별 로직: 상품명 앞부분의 접두사("삼성", "미래에셋" 등)를 정규식이나 매핑 테이블로 분석하여 금융회사를 분리합니다.

### **5.2 AI 프롬프트 생성 모듈 (고도화)**

사용자 질의와 설정값을 종합하여 체계적인 프롬프트를 만듭니다.

1. System 지시문: "당신은 퇴직연금(IRP) 포트폴리오 전문가입니다."  
2. 계좌 및 상품 유니버스 주입: 파싱된 CSV 데이터와 화면 3의 보유 상품 리스트 주입.  
3. 매도 및 시점 제약 조건 주입 (중요):  
   * 화면 4에서 선택한 값에 따라 "현재 시점의 즉각적인 리밸런싱을 제안할 것" 또는 "만기 도래 시점을 가정한 예약용 리밸런싱을 제안할 것"이라는 상황을 명시합니다.  
   * "보유 상품 중 A는 전량 매도, B는 X원 부분 매도, C는 유지하도록 사용자가 결정했으니, AI는 이에 따라 확보된 총 현금(매도 대금 합계)을 바탕으로만 신규 매수 포트폴리오를 구성할 것"이라는 강제 조건을 추가합니다.  
4. 운용사 제약 조건: 화면 1에서 "종신형 연금"을 체크한 경우 "추천 펀드는 운용사가 '삼성생명'인 상품만으로 제한"하도록 텍스트를 추가합니다.

### **5.3 PDF 생성 로직 (QuestPDF 활용)**

* QuestPDF의 Fluent API를 활용해 생성합니다.  
* 헤더에 타이틀 및 리밸런싱 성격(즉시 vs 만기예약)을 표기하고, 사용자의 매도/유지 결정 내역표를 먼저 그린 뒤, AI의 마크다운 응답을 텍스트 블록으로 파싱하여 삽입합니다.

## **6\. 개발 진행 순서 (권장)**

1. Phase 1 (UI 스캐폴딩): WinUI 3 프로젝트 생성 및 NavigationView 기반 5개 탭 레이아웃 뼈대 구축.  
2. Phase 2 (데이터 파싱): HTML 로드 및 CSV 변환, 정규식을 이용한 운용사 추출 로직 구현.  
3. Phase 3 (계좌 및 매도 UI): 화면 3과 화면 4의 DataGrid 연동. 매도 방식(전량/부분/유지)에 따른 텍스트 박스 활성화/비활성화 데이터 바인딩(MVVM) 구현.  
4. Phase 4 (AI 통신): 매도 설정 조건과 추가 질의를 조합하는 프롬프트 빌더 작성 및 HttpClient 통신 로직 완성.  
5. Phase 5 (PDF 출력): QuestPDF 연동 및 레이아웃 템플릿 완성, 최종 테스트.
