# 연금나침반 v1.3.0

삼성생명 IRP(개인형 퇴직연금) 계좌의 펀드/원리금보장형 비중 조정을 AI(Claude·Gemini·GPT)의 도움으로 계획하기 위한 Windows 데스크톱 도구입니다.

> ⚠️ **본 앱은 삼성생명보험주식회사와 무관한 비공식(third-party) 도구이며, AI가 생성한 제안은 참고용 정보일 뿐 투자 권유가 아닙니다.** 모든 투자 결정과 그 결과에 대한 책임은 사용자 본인에게 있습니다.

## v1.3.0 변경 사항 — 클라우드 API Key 암호화 (동기화 암호구문)

v1.2.0에서 API Key를 Google 계정(drive.appdata)에 동기화하는 기능이 추가됐지만, 키가 **평문**으로 올라갔습니다. v1.3.0은 이 키를 **사용자 암호구문으로 암호화**한 뒤에만 클라우드에 저장하도록 강화합니다. 혹시 다른 사람이 쓰거나 Google 계정이 노출되는 상황까지 고려한 보안 강화 회차입니다.

### 새 기능 — 동기화 암호구문 (PBKDF2 + AES-256-GCM)

환경 설정 → 동기화 → Google 계정 연동 카드에 **"동기화 암호구문"** 입력란이 생겼습니다.

- **암호화 방식:** 암호구문에서 키를 유도(PBKDF2-HMAC-SHA256, 60만 회)한 뒤 **AES-256-GCM**(인증 암호화)으로 `apikeys.json`을 암호화. .NET 내장 암호 모듈만 사용합니다.
- **암호구문은 이 PC에만 저장됩니다.** Windows 자격증명 보관소에만 보관되고 클라우드로는 절대 전송되지 않습니다. 따라서 **강력한 암호구문을 쓰는 한, Google 계정이 노출되더라도 키 자체는 복호화되지 않습니다.**
- **새 PC에서는 잠금 해제 1회.** 다른 PC에서 설정한 암호구문을 입력하면(Enter 또는 버튼) 클라우드의 키를 풀어 불러옵니다. 한 PC당 한 번만 입력하면 됩니다.
- **로컬 전용 사용자는 영향 없음.** Google 동기화를 쓰지 않으면 암호구문을 볼 일도 없습니다(매 실행 입력 같은 마찰 없음).

### 보안 설계 — 적대적 리뷰로 다듬은 자세

다중 에이전트 적대적 보안 리뷰를 거쳐 다음을 보강했습니다:

- **평문 다운그레이드 차단:** 이미 암호구문을 쓰는 기기는 클라우드의 비암호화(평문) 키 파일을 **거부**합니다. 클라우드 파일을 조작해 암호화를 벗기거나 공격자 키를 주입하려는 시도를 막습니다. (v1.2.0 평문 키는 암호구문 미설정 신규 기기에서 1회만 불러오고, 곧바로 암호화 설정을 안내합니다.)
- **약한 암호구문 방지:** 설정 시 최소 길이(8자, 12자+ 권장)와 **확인 재입력**을 요구합니다(오타로 영구 잠김 방지).
- **덮어쓰기 보호:** 네트워크 일시 오류로 클라우드 키를 못 읽었을 때, 진짜 키 파일을 빈 값으로 덮어쓰지 않습니다.
- **손상 복구:** 클라우드 키 파일이 손상되면 현재 키로 다시 만들 수 있게 안내합니다.
- **조작된 파라미터 방어:** 암호 봉투의 반복 횟수를 정상 범위로 제한해 CPU 고갈/강도 약화 조작을 막습니다.

### 부수 수정

- Google 연결/잠금 해제 직후 사고 수준·AI 공급자 라디오 선택이 간헐적으로 초기화되던 문제를 수정했습니다(WinUI RadioButtons 되돌림 버그 방어).
- 암호구문 입력란에서 **Enter**로 바로 적용/잠금 해제할 수 있습니다.

### 함께 정리한 문서

- **개인정보 처리방침** 추가([PRIVACY.md](../../PRIVACY.md)) — 데이터 흐름(개발자 무수집, drive.appdata 한정, AI 공급자 전송)을 명시. OAuth 동의 화면 신뢰도용.
- OAuth 브랜딩/검증 안내([doc/google-oauth-branding-verification.md](../../doc/google-oauth-branding-verification.md)) — drive.appdata는 non-sensitive라 정식 검증 불필요라는 정리 포함.

### 호환성

- v1.2.0 → v1.3.0 업그레이드: 로컬 키는 그대로. Google 동기화를 쓰던 사용자는 클라우드의 평문 키가 그대로 로드되며, **암호구문을 설정하는 순간 암호화되어 다시 업로드**됩니다. 설정 전까지 새로 입력한 키는 이 PC에만 저장됩니다.
- 암호구문을 잊으면 클라우드 키는 복구할 수 없습니다(설계상 보안 속성). 이 경우 키를 다시 입력하고 새 암호구문을 설정하면 됩니다.

## 설치 방법

다음 두 파일을 함께 받으세요:

| 파일 | 용도 |
| --- | --- |
| `PensionCompass_1.3.0.0_x64.msixbundle` | 앱 본체 |
| `PensionCompass_1.3.0.0_x64.cer` | self-signed 코드 서명 인증서 (기존과 동일) |

### 1단계: 인증서 신뢰 등록 (PC당 최초 1회)

이미 v1.0.x~v1.2.x 인증서를 등록해 두셨다면 생략 가능합니다 (인증서 동일).

자체 서명 인증서이므로 받는 PC가 발급자(CN=Changhwan Bang)를 신뢰하도록 등록해야 합니다. PowerShell 한 줄(관리자):

```pwsh
Import-Certificate -FilePath ".\PensionCompass_1.3.0.0_x64.cer" `
                   -CertStoreLocation Cert:\LocalMachine\Root
```

> 자체 서명 인증서는 그 자체가 루트라 "신뢰할 수 있는 게시자"/"신뢰할 수 있는 사람"에 넣으면 체인 검증이 끊겨 `0x800B010A`(CERT_E_CHAINING)로 설치가 실패합니다. 반드시 **로컬 컴퓨터 → 신뢰할 수 있는 루트 인증 기관** 조합이어야 합니다.

### 2단계: 앱 설치

`PensionCompass_1.3.0.0_x64.msixbundle` 더블클릭 → "설치". 기존 버전이 있으면 in-place 업데이트되며 데이터·동기 설정은 유지됩니다.

### 3단계: 동기화 암호구문 설정 (Google 동기화 사용 시)

환경 설정 → 동기화 → Google 계정 연동 → 연결 후 "동기화 암호구문"에 길고 추측하기 어려운 암호구문을 입력하고 적용. 다른 PC에서는 같은 암호구문으로 잠금 해제하면 키가 따라옵니다.

## 시스템 요구 사항

- Windows 10 17763(1809) 이상 / Windows 11
- x64 아키텍처
- AI 호출용 인터넷 연결 + 사용자 본인의 API Key (Anthropic Claude / Google Gemini / OpenAI 중 하나 이상)
- (선택) Google Drive 연동 사용 시 Google 계정

## 데이터 보관

- **계좌·카탈로그·이력**: 사용자 PC의 LocalState + (Google 연동 시) drive.appdata 양쪽 미러
- **API Key — 로컬**: Windows 자격증명 보관소(`PensionCompass.ApiKey`), 클라우드 미전송
- **API Key — 클라우드 (Google 연결 시)**: 자격증명 보관소 캐시(`PensionCompass.ApiKey.Cloud`) + drive.appdata의 `apikeys.json`. **v1.3.0부터 암호구문으로 암호화(AES-256-GCM)되어 저장**
- **동기화 암호구문**: 자격증명 보관소(`PensionCompass.SyncPassphrase`)에 이 PC에만 보관, 클라우드 미전송
- **AI 호출 시**: 보유 현황과 카탈로그가 사용자가 선택한 AI 공급자에 직접 전송. 그 외 외부 전송 없음
