# 연금나침반 v1.2.0

삼성생명 IRP(개인형 퇴직연금) 계좌의 펀드/원리금보장형 비중 조정을 AI(Claude·Gemini·GPT)의 도움으로 계획하기 위한 Windows 데스크톱 도구입니다.

> ⚠️ **본 앱은 삼성생명보험주식회사와 무관한 비공식(third-party) 도구이며, AI가 생성한 제안은 참고용 정보일 뿐 투자 권유가 아닙니다.** 모든 투자 결정과 그 결과에 대한 책임은 사용자 본인에게 있습니다.

## v1.2.0 변경 사항 — Google 계정에 API Key 동기화 + 커버드콜 펀드 제외 옵션

이번 회차는 두 가지 사용자 요청 반영:

1. **Google 계정에 API Key 연동** — 다른 PC에서 Google 로그인만 하면 API Key까지 자동으로 따라옵니다.
2. **커버드콜 펀드 추천 제외 옵션** — 변동성 프로파일이 마음에 안 드는 사용자가 후보군에서 통째로 빼고 추천을 받을 수 있게 됐습니다.

### 새 기능 ① — Google 계정과 API Key 연동

기존 v1.1.0까지는 계좌·카탈로그·이력만 Google Drive로 자동 동기되고, API Key는 PC별로 다시 입력해야 했습니다. v1.2.0부터는 **로그인 = 클라우드 키, 로그아웃 = 로컬 키**로 활성 출처가 명확하게 분리됩니다.

#### 동작 방식 (두 저장소, 자동 섞이지 않음)

| 상황 | 활성 API Key 출처 | 변경 시 영향 |
| --- | --- | --- |
| Google 미연결 / 연결 해제 | PC의 로컬 PasswordVault (오프라인 키) | 로컬에만 저장 |
| Google 연결됨 | Drive의 키 (Google 계정에 묶임) | 즉시 Drive로 업로드 |
| Google 연결 후 다른 PC로 이동 | Drive에서 자동 다운로드 | 다른 PC에서도 동일 키 사용 |

- **두 저장소가 절대 자동으로 섞이지 않습니다.** Google 연결 시 로컬 키를 덮어쓰지 않고, 연결 해제 시 Drive의 키를 PC로 가져오지 않습니다. 두 세트의 키를 독립적으로 유지하고 싶다면 그대로 가능합니다.
- **신규 PC + Google 연결 = 빈 키부터 시작.** 새 PC에서 Google 연결을 처음 했을 때 Drive에 키가 없으면 입력란이 비어 있습니다. 직접 다시 입력해 주세요 — 이후 자동으로 다른 PC들과 공유됩니다.
- **PC 분실 / OAuth revoke 시 캐시 정리.** "연결 해제" 버튼을 누르면 PC 안의 클라우드 캐시 슬롯이 즉시 비워집니다.

#### 저장 위치 (보안)

- **로컬 키 (오프라인용):** Windows 자격증명 보관소의 `PensionCompass.ApiKey` 자원 — v1.0.4 이래 그대로
- **클라우드 캐시 (연결 시 사용):** Windows 자격증명 보관소의 `PensionCompass.ApiKey.Cloud` 자원 (신규) + Google Drive 앱 전용 폴더의 `apikeys.json` (drive.appdata)
- drive.appdata는 본 앱의 OAuth 클라이언트 ID + 본인 Google 계정 조합에서만 접근 가능하며 Drive 웹 UI에서는 보이지 않습니다. 그 외 제3자 서비스로의 전송은 없습니다.

### 새 기능 ② — 커버드콜 펀드 추천 제외 옵션

내 계좌 화면의 가입자 정보 섹션에 체크박스 한 줄이 추가됐습니다:

> ☐ **커버드콜 펀드 추천에서 제외** — 상품명에 "커버드콜"이 포함된 펀드를 매수 후보에서 제거합니다. 이미 보유 중인 커버드콜 펀드의 매도·유지는 위의 매도 정책으로 따로 결정됩니다.

체크 시 AI에게 전달되는 카탈로그에서 커버드콜 전략 펀드(예: `[온라인전용]미래에셋퇴직연금배당커버드콜액티브증권자1`, `[온라인]신한퇴직연금커버드콜인덱스증권자투자신탁` 등)가 **사전에 통째로 제거**됩니다. AI가 실수로라도 추천 후보로 꺼낼 가능성이 없습니다. 보유 중인 커버드콜 펀드는 보유 상품 섹션에 그대로 두며, 매도 여부는 사용자가 매도 가능/금지 체크박스로 결정합니다.

식별 기준은 단순 키워드 매칭(`커버드콜` / `Covered Call` / `CoveredCall` — 대소문자 무시). 한국 퇴직연금 펀드는 커버드콜 전략을 상품명에 명시적으로 표기하므로 신뢰성 있게 동작합니다.

### 호환성

- v1.1.0 → v1.2.0 업그레이드 사용자: 기존 로컬 PasswordVault 슬롯은 그대로 유지. Google 미연결 상태에서는 v1.1.0과 동일하게 동작.
- Google에 이미 연결된 상태로 v1.2.0을 처음 켜면 Drive에 `apikeys.json`이 아직 없으므로 입력란이 비어 보일 수 있습니다. 다시 한 번 입력하면 그 때부터 Drive에 저장되어 다른 PC와 자동 공유됩니다.
- 커버드콜 제외 옵션은 기본값 OFF — 명시적으로 체크해야 적용됩니다.

### 1단계: 인증서 신뢰 등록 (PC당 최초 1회)

이미 v1.0.x / v1.1.0 인증서를 등록해 두셨다면 이 단계는 생략 가능합니다 (인증서 자체는 동일).

자체 서명 인증서이므로 받는 PC가 발급자(CN=Changhwan Bang)를 신뢰하도록 등록해야 합니다.

1. `PensionCompass_1.2.0.0_x64.cer` 더블클릭
2. **인증서 설치(I)** 버튼 클릭
3. 저장소 위치: **로컬 컴퓨터(L)** 선택 → 다음 (UAC 동의 필요)
4. 인증서 저장소: **모든 인증서를 다음 저장소에 저장(P)** → 찾아보기 → **신뢰할 수 있는 루트 인증 기관** 선택 → 확인 → 다음 → 마침

> 자체 서명 인증서는 그 자체가 루트이기 때문에 "신뢰할 수 있는 게시자" 나 "신뢰할 수 있는 사람" 에 넣으면 체인 검증이 끊겨 `0x800B010A` (CERT_E_CHAINING) 오류로 설치가 실패합니다. 반드시 **로컬 컴퓨터 → 신뢰할 수 있는 루트 인증 기관** 조합이어야 합니다.

PowerShell 한 줄로도 가능 (관리자):

```pwsh
Import-Certificate -FilePath ".\PensionCompass_1.2.0.0_x64.cer" `
                   -CertStoreLocation Cert:\LocalMachine\Root
```

### 2단계: 앱 설치

`PensionCompass_1.2.0.0_x64.msixbundle` 더블클릭 → "설치" 버튼.

기존 v1.1.0이 설치돼 있다면 in-place 업데이트되며 기존 데이터는 그대로 유지됩니다.

### 3단계: Google 연결 (선택)

- 환경 설정 → "동기화" 섹션 → **Google 계정 연동** 라디오 선택 → "Google 계정 연결" 클릭
- v1.2.0에서 자동으로 API Key 동기화까지 함께 처리됩니다 (별도 토글 없음)
- 자세한 절차는 [v1.1.0 릴리스 노트](v1.1.0.md)의 "3단계: Google 연결" 섹션 참조

- x64 아키텍처

## 데이터 보관

- **계좌·카탈로그·이력**: 사용자 PC의 앱 LocalState 폴더 + (Google 연동 시) Google Drive `drive.appdata` 양쪽에 미러 저장. JSON 평문이며 사용자 본인의 Google 계정 + 본 앱 OAuth 클라이언트 조합에서만 접근 가능
- **API Key — 로컬**: Windows 자격증명 보관소 (`PensionCompass.ApiKey`)에 사용자 계정으로 암호화 보관. 클라우드에 안 올라감
- **API Key — 클라우드 (v1.2.0+, Google 연결 시)**: Windows 자격증명 보관소 (`PensionCompass.ApiKey.Cloud`)에 캐시 + Google Drive `drive.appdata`의 `apikeys.json`에 평문 동기. 로컬 키와 절대 섞이지 않음
- **OAuth 토큰**: Windows 자격증명 보관소에 access/refresh token 보관, 자동 갱신
- **AI 호출 시**: 보유 현황과 카탈로그가 사용자가 선택한 AI 공급자(Anthropic / Google / OpenAI)에 직접 전송됩니다. 그 외 외부 전송은 없습니다
