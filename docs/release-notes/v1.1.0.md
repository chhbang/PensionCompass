# 연금나침반 v1.1.0

삼성생명 IRP(개인형 퇴직연금) 계좌의 펀드/원리금보장형 비중 조정을 AI(Claude·Gemini·GPT)의 도움으로 계획하기 위한 Windows 데스크톱 도구입니다.

> ⚠️ **본 앱은 삼성생명보험주식회사와 무관한 비공식(third-party) 도구이며, AI가 생성한 제안은 참고용 정보일 뿐 투자 권유가 아닙니다.** 모든 투자 결정과 그 결과에 대한 책임은 사용자 본인에게 있습니다.

## v1.1.0 변경 사항 — 클라우드 직접 연동

이번 회차의 큰 변화: **앱 안에서 Google 계정으로 로그인 한 번이면 다른 PC에서도 자동으로 데이터를 따라옵니다.** v1.0.x의 "사용자가 직접 클라우드 동기 폴더를 만들고 그 경로를 앱에 적어넣기" 방식은 셋업이 번거롭고 PC가 바뀔 때마다 같은 설정을 반복해야 했는데, 이제는 Google OAuth 한 번이면 끝입니다.

### 새 기능: Google Drive 직접 연동 (drive.appdata)

- 환경 설정에 **"동기화"** 라디오 (안 함 / 폴더 동기 (기존) / **Google 계정 연동**) 추가
- "Google 계정 연결" 버튼 클릭 → 기본 브라우저가 열리며 본인 Gmail로 로그인 + 권한 승인 → 자동으로 앱이 연결 완료
- 데이터는 **Google Drive의 숨겨진 앱 전용 폴더(`drive.appdata`)**에 저장됨
  - Drive 웹 UI에선 안 보이는 영역이라 가입자 잔고 정보가 사용자 본인 Drive 메인에 노출되지 않음
  - 본 앱의 OAuth 클라이언트 ID + 본인 Google 계정 조합에서만 접근 가능
- 다른 PC에서 같은 Google 계정으로 연결하면 자동으로 클라우드 데이터를 받아옴 — 폴더 경로 입력 같은 건 묻지 않음
- **첫 연결 시 마이그레이션 자동 처리** — 기존 LocalState에 데이터가 있고 Drive가 비어 있으면 자동으로 한 번 업로드. 사용자 추가 행동 불필요

### 아키텍처: 동기화 공급자 추상화

내부적으로 `ISyncProvider` 인터페이스를 도입했고, 향후 OneDrive·Dropbox 같은 다른 클라우드 공급자를 추가하기 용이한 구조가 되었습니다. v1.1.0에선 다음 3개 구현체:

- **GoogleDriveSyncProvider** (신규, Google Drive `drive.appdata`)
- **FilesystemFolderSyncProvider** (v1.0.x 폴더 동기 방식, 호환을 위해 유지)
- **NoopSyncProvider** (동기 미설정 시)

`StateStore`는 어느 공급자가 백엔드인지 모르고 동일한 인터페이스 뒤에서 동작합니다. 사용자가 환경 설정에서 동기 모드를 바꾸면 앱 재시작 없이 즉시 적용됩니다.

### 변경 없는 부분 (호환성)

- **v1.0.7 사용자가 v1.1.0으로 업그레이드** → 기존 동기 폴더 설정이 자동으로 "폴더 동기" 모드로 인식되어 동작 그대로 유지. 사용자가 Google 모드로 바꾸고 싶을 때만 환경 설정에서 라디오를 옮기면 됨
- **API Key 보관** — 여전히 Windows 자격증명 보관소(`PasswordVault`)에 사용자 계정으로 암호화 보관. 클라우드에는 절대 안 올라감
- **임포트한 원본 HTML** — 가입자 성명·계좌번호 포함 가능하므로 여전히 저장 안 함

## 설치 방법

다음 두 파일을 함께 받으세요:

| 파일 | 용도 |
| --- | --- |
| `PensionCompass_1.1.0.0_x64.msixbundle` | 앱 본체 |
| `PensionCompass_1.1.0.0_x64.cer` | self-signed 코드 서명 인증서 |

### 1단계: 인증서 신뢰 등록 (PC당 최초 1회)

자체 서명 인증서이므로 받는 PC가 발급자(CN=Changhwan Bang)를 신뢰하도록 등록해야 합니다.

1. `PensionCompass_1.1.0.0_x64.cer` 더블클릭
2. **인증서 설치(I)** 버튼 클릭
3. 저장소 위치: **로컬 컴퓨터(L)** 선택 → 다음 (UAC 동의 필요)
4. 인증서 저장소: **모든 인증서를 다음 저장소에 저장(P)** → 찾아보기 → **신뢰할 수 있는 루트 인증 기관** 선택 → 확인 → 다음 → 마침

> 자체 서명 인증서는 그 자체가 루트이기 때문에 "신뢰할 수 있는 게시자" 나 "신뢰할 수 있는 사람" 에 넣으면 체인 검증이 끊겨 `0x800B010A` (CERT_E_CHAINING) 오류로 설치가 실패합니다. 반드시 **로컬 컴퓨터 → 신뢰할 수 있는 루트 인증 기관** 조합이어야 합니다.

PowerShell 한 줄로도 가능 (관리자):

```pwsh
Import-Certificate -FilePath ".\PensionCompass_1.1.0.0_x64.cer" `
                   -CertStoreLocation Cert:\LocalMachine\Root
```

### 2단계: 앱 설치

`PensionCompass_1.1.0.0_x64.msixbundle` 더블클릭 → "설치" 버튼.

설치 완료 후 시작 메뉴에서 "연금나침반"으로 검색해 실행할 수 있습니다.

### 3단계: Google 연결 (선택)

다른 PC와 자동 동기를 쓰시려면:

1. 환경 설정 → "동기화" 섹션 → **Google 계정 연동** 라디오 선택
2. "Google 계정 연결" 버튼 클릭
3. 브라우저가 열림 → 본인 Gmail로 로그인
4. "Google에서 이 앱을 인증하지 않았습니다" 경고 화면이 뜨면 **고급(Advanced)** → **PensionCompass(unsafe)로 이동(Continue)** 클릭 (자체 OAuth 앱이라 정상)
5. 권한 승인 → **계속(Continue)** / **허용(Allow)** — `drive.appdata` 권한만 요청됩니다
6. 앱 화면에 "✓ user@example.com 연결됨" 표시되면 끝

다른 PC에서도 같은 절차 반복. 데이터는 자동으로 따라옵니다.

### 업데이트하는 경우

기존 버전이 이미 설치되어 있다면 새 .msixbundle을 더블클릭하면 in-place 업데이트됩니다 (인증서 재등록 불필요, 기존 입력 데이터 유지). 기존 폴더 동기화 설정도 그대로 보존되며, "Google 계정 연동"으로 옮기시면 첫 연결 시 자동으로 마이그레이션 됩니다.

## 시스템 요구 사항

- Windows 10 17763(1809) 이상 / Windows 11
- x64 아키텍처
- AI 호출용 인터넷 연결 + 사용자 본인의 API Key (Anthropic Claude / Google Gemini / OpenAI 중 하나 이상)
- (선택) Google Drive 연동 사용 시 Google 계정

## 데이터 보관

- **계좌·카탈로그·이력**: 사용자 PC의 앱 LocalState 폴더 + (Google 연동 시) Google Drive `drive.appdata` 양쪽에 미러 저장. JSON 평문이며 사용자 본인의 Google 계정 + 본 앱 OAuth 클라이언트 조합에서만 접근 가능
- **API Key 3개**: Windows 자격증명 보관소 (사용자 계정으로 암호화). **클라우드에 절대 기록되지 않음** — PC를 옮기시면 API 키는 재입력 필요 (v1.2.0+에서 옵션으로 클라우드 동기화 검토 예정)
- **OAuth 토큰**: Windows 자격증명 보관소에 access/refresh token 보관, 자동 갱신
- **AI 호출 시**: 보유 현황과 카탈로그가 사용자가 선택한 AI 공급자(Anthropic / Google / OpenAI)에 직접 전송됩니다. 그 외 외부 전송은 없습니다
