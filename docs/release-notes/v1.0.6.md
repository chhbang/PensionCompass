# 연금나침반 v1.0.6

삼성생명 IRP(개인형 퇴직연금) 계좌의 펀드/원리금보장형 비중 조정을 AI(Claude·Gemini·GPT)의 도움으로 계획하기 위한 Windows 데스크톱 도구입니다.

> ⚠️ **본 앱은 삼성생명보험주식회사와 무관한 비공식(third-party) 도구이며, AI가 생성한 제안은 참고용 정보일 뿐 투자 권유가 아닙니다.** 모든 투자 결정과 그 결과에 대한 책임은 사용자 본인에게 있습니다.

## v1.0.6 변경 사항

### 새 기능: IRP 70/30 자산 비중 규제 자동 검증

한국 IRP 제도는 법적으로 **위험자산 비중 70% 상한 / 안정자산 30% 의무**가 적용됩니다. 이 룰을 위반한 추천은 삼성생명 시스템에서 거부되어 사용자가 다시 짜야 했는데, v1.0.6부터는 AI 응답을 받자마자 앱이 자동으로 비중을 검증하고 위반 시 즉시 경고합니다.

- **AI 프롬프트에 IRP 70/30 제약을 명시적 hard constraint로 추가** — AI가 추천 시 자산구분별 합계 비중을 응답에 명시적으로 포함하도록 요구
- **AI 리밸런싱 화면에 검증 결과 배너 표시** — 색상으로 한눈에 인지:
  - 초록 — 70/30 준수, 추출된 위험자산·안정자산 비중 표시
  - 빨강 — 한도 위반 감지, 어느 쪽이 어떻게 위반인지 표시
  - 노랑 — AI 응답에서 비중 자동 추출 실패, 직접 확인 필요
- **검증 로직은 결정론적** — AI 판단에 의존하지 않고 마크다운 응답에서 비중 숫자를 정규식으로 추출해 한도와 비교

### 개선: 펀드 카탈로그에 자산구분 자동 분류

삼성생명 상품 페이지의 위험자산 뱃지를 파서가 인식해, 펀드별로 **위험자산 / 안정자산** 분류를 카탈로그에 보존하게 했습니다. 이 분류가 위 70/30 검증의 결정론적 근거가 됩니다.

- HTML 파서가 펀드 카드의 `위험자산` 뱃지를 인식 — 뱃지가 없으면 안정자산으로 이진 추론 (채권형·머니마켓형 등 안전 펀드 정확 분류)
- AI 프롬프트의 펀드 표에 **자산구분 컬럼 추가** — AI가 추천을 짤 때 각 펀드의 분류를 직접 보고 결정
- 펀드 CSV 라운드트립에도 `자산구분` 컬럼 포함 — 옛 CSV(컬럼 없는 v1.0.5 이전 export)는 빈 값으로 fallback되며 정상 로드
- 원리금보장 상품은 정의상 안정자산 — 프롬프트가 이 사실을 AI에게 명시

## 설치 방법

다음 두 파일을 함께 받으세요:

| 파일 | 용도 |
| --- | --- |
| `PensionCompass_1.0.6.0_x64.msixbundle` | 앱 본체 |
| `PensionCompass_1.0.6.0_x64.cer` | self-signed 코드 서명 인증서 |

### 1단계: 인증서 신뢰 등록 (PC당 최초 1회)

자체 서명 인증서이므로 받는 PC가 발급자(CN=Changhwan Bang)를 신뢰하도록 등록해야 합니다.

1. `PensionCompass_1.0.6.0_x64.cer` 더블클릭
2. **인증서 설치(I)** 버튼 클릭
3. 저장소 위치: **로컬 컴퓨터(L)** 선택 → 다음 (UAC 동의 필요)
4. 인증서 저장소: **모든 인증서를 다음 저장소에 저장(P)** → 찾아보기 → **신뢰할 수 있는 루트 인증 기관** 선택 → 확인 → 다음 → 마침

> 자체 서명 인증서는 그 자체가 루트이기 때문에 "신뢰할 수 있는 게시자" 나 "신뢰할 수 있는 사람" 에 넣으면 체인 검증이 끊겨 `0x800B010A` (CERT_E_CHAINING) 오류로 설치가 실패합니다. 반드시 **로컬 컴퓨터 → 신뢰할 수 있는 루트 인증 기관** 조합이어야 합니다.

PowerShell 한 줄로도 가능 (관리자):

```pwsh
Import-Certificate -FilePath ".\PensionCompass_1.0.6.0_x64.cer" `
                   -CertStoreLocation Cert:\LocalMachine\Root
```

### 2단계: 앱 설치

`PensionCompass_1.0.6.0_x64.msixbundle` 더블클릭 → "설치" 버튼.

설치 완료 후 시작 메뉴에서 "연금나침반"으로 검색해 실행할 수 있습니다.

### 업데이트하는 경우

기존 버전이 이미 설치되어 있다면 새 .msixbundle을 더블클릭하면 in-place 업데이트됩니다 (인증서 재등록 불필요, 기존 입력 데이터 유지). 동기화 폴더 설정도 그대로 유지됩니다.

## 시스템 요구 사항

- Windows 10 17763(1809) 이상 / Windows 11
- x64 아키텍처
- AI 호출용 인터넷 연결 + 사용자 본인의 API Key (Anthropic Claude / Google Gemini / OpenAI 중 하나 이상)

## 데이터 보관

- **계좌·카탈로그·이력**: 사용자 PC의 앱 LocalState 폴더 + (지정 시) 동기화 폴더 양쪽에 미러 저장. JSON 평문이며 사용자 본인의 PC/클라우드 계정 안에서만 보입니다
- **API Key 3개**: Windows 자격증명 보관소 (사용자 계정으로 암호화). 동기화 폴더에 절대 기록되지 않음
- **AI 호출 시**: 보유 현황과 카탈로그가 사용자가 선택한 AI 공급자(Anthropic / Google / OpenAI)에 직접 전송됩니다. 그 외 외부 전송은 없습니다
