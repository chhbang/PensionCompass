# 연금나침반 v1.4.0

삼성생명 IRP(개인형 퇴직연금) 계좌의 펀드/원리금보장형 비중 조정을 AI(Claude·Gemini·GPT)의 도움으로 계획하기 위한 Windows 데스크톱 도구입니다.

> ⚠️ **본 앱은 삼성생명보험주식회사와 무관한 비공식(third-party) 도구이며, AI가 생성한 제안은 참고용 정보일 뿐 투자 권유가 아닙니다.** 모든 투자 결정과 그 결과에 대한 책임은 사용자 본인에게 있습니다.

## v1.4.0 변경 사항 — PDF 참고자료 첨부 + 크로스-PC 동기화 완성

두 갈래의 큰 변화입니다: ① AI에게 운용사 가이드북·증권사 리포트 같은 **PDF 참고자료를 분류해서 함께** 전달, ② Google 계정으로 **"이 PC에서 하던 작업을 저 PC에서 그대로"** 이어받도록 동기화 범위를 대폭 넓혔습니다.

### 새 기능 ① — PDF 참고자료 첨부 (분류 포함)

새 화면 **"참고 자료 (PDF)"** 가 추가됐습니다. 자산운용사 펀드/ETF 가이드, 증권사 시장 분석 리포트 등 PDF를 등록해 두면 AI 리밸런싱 요청 시 함께 전달됩니다.

- **자료 분류**: 등록 시 분류를 지정합니다 — **펀드/ETF 가이드**(상품 이해용) / **시장 분석 리포트**(시장 환경 판단용) / **기타**. 분류별 활용 지침이 프롬프트에 함께 전달되어 AI가 용도에 맞게 사용합니다.
- **편향 가드레일**: "특정 운용사·종목을 편애하지 말고, 최종 결정은 카탈로그 수치로" 라는 안내가 자동 포함됩니다.
- **공급자별 전송**: Claude·GPT는 inline(base64, ~32MB 한도 가드), **Gemini는 File API**(업로드→처리→참조)로 대용량 PDF도 지원합니다.
- 각 자료는 **"AI에 첨부"** 토글로 이번 요청에 포함할지 제어합니다.

### 새 기능 ② — 크로스-PC 동기화 완성

Google 계정 연동의 목적은 "다른 PC에서도 이어서 작업"입니다. v1.4.0에서 동기화 범위를 넓히고 로그인 시 자동 로드 버그를 고쳤습니다.

| 데이터 | v1.3.x | v1.4.0 |
| --- | --- | --- |
| API Key (암호화) | ✅ | ✅ |
| 계좌·보유상품 / 상품 카탈로그 | 일부 (연결 시 다운로드 누락) | ✅ **연결 즉시 클라우드에서 로드** (버그 수정) |
| 선택 AI·모델·사고 수준 | ❌ | ✅ **동기화** (`settings.json`) |
| 리밸런싱 이력 | ❌ (PC 간 공유 안 됨) | ✅ **동기화** |
| 참고자료 PDF | — | ✅ **문서별 opt-in 동기화** |

- **로그인 시 자동 로드 수정**: 기존엔 Google 연결 후에도 Drive의 계좌/카탈로그를 앱으로 내려받지 않아, 새 PC(또는 재설치)에서 로그인해도 비어 보였습니다. 이제 연결 즉시 클라우드 데이터를 불러옵니다("cloud-as-source-of-truth").
- **환경설정 동기화**: 한 PC에서 공급자를 Gemini로 바꾸면 다른 PC에서도 따라옵니다.
- **이력 동기화**: 한 PC에서 받은 추천이 다른 PC의 이력 화면에 나타납니다.
- **PDF 참고자료 동기화 — 용량 고려한 opt-in**: PDF는 용량이 커서 Google 계정 저장공간을 사용하므로 **기본은 꺼짐**. 각 자료의 **"클라우드"** 체크박스로 켠 자료만 업로드됩니다. 화면에 동기 용량 합계와 안내가 표시됩니다. 업로드는 백그라운드 best-effort이며, 저장공간이 부족하면 다른 PC에 안 나타날 뿐 로컬 원본은 안전합니다.

### 부수 수정

- Google 연결/잠금 해제 직후 AI 공급자·사고 수준 라디오 선택이 간헐적으로 초기화되던 버그 수정(WinUI RadioButtons -1 되돌림 방어).
- 동기화 암호구문 입력란에서 **Enter**로 바로 적용/잠금 해제.

### 호환성

- v1.3.x → v1.4.0: in-place 업데이트, 기존 데이터·동기 설정 유지. 이력/환경설정/계좌/카탈로그는 Google 연결 시 자동으로 클라우드와 맞춰집니다.
- PDF 참고자료 클라우드 동기화는 기본 꺼짐 — 명시적으로 켜야 업로드됩니다.

## 설치 방법

| 파일 | 용도 |
| --- | --- |
| `PensionCompass_1.4.0.0_x64.msixbundle` | 앱 본체 |
| `PensionCompass_1.4.0.0_x64.cer` | self-signed 코드 서명 인증서 (기존과 동일) |

### 1단계: 인증서 신뢰 등록 (PC당 최초 1회)

이미 v1.0.x~v1.3.x 인증서를 등록해 두셨다면 생략 가능합니다 (인증서 동일). PowerShell 한 줄(관리자):

```pwsh
Import-Certificate -FilePath ".\PensionCompass_1.4.0.0_x64.cer" `
                   -CertStoreLocation Cert:\LocalMachine\Root
```

> 자체 서명 인증서는 그 자체가 루트라 "신뢰할 수 있는 게시자"/"신뢰할 수 있는 사람"에 넣으면 `0x800B010A`(CERT_E_CHAINING)로 실패합니다. 반드시 **로컬 컴퓨터 → 신뢰할 수 있는 루트 인증 기관** 조합이어야 합니다.

### 2단계: 앱 설치

`PensionCompass_1.4.0.0_x64.msixbundle` 더블클릭 → "설치". 기존 버전이 있으면 in-place 업데이트되며 데이터·동기 설정은 유지됩니다.

## 시스템 요구 사항

- Windows 10 17763(1809) 이상 / Windows 11, x64
- AI 호출용 인터넷 연결 + 사용자 본인의 API Key (Anthropic Claude / Google Gemini / OpenAI 중 하나 이상)
- (선택) Google Drive 연동 사용 시 Google 계정

## 데이터 보관 (요약)

- **계좌·카탈로그·이력·환경설정**: LocalState + (Google 연동 시) drive.appdata 미러
- **API Key**: 로컬 PasswordVault, 클라우드엔 암호구문으로 암호화(AES-256-GCM)된 형태로만
- **참고자료 PDF**: LocalState + (각 자료 opt-in 시) drive.appdata
- **AI 호출 시**: 보유 현황·카탈로그·첨부 PDF가 사용자가 선택한 AI 공급자에 직접 전송. 그 외 외부 전송 없음
